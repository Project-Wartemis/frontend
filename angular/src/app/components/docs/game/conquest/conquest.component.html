1. Mechanics

a. Start

The game is played on a graph. There are a set number of nodes, connected by links.
Each node is controlled by a player (possibly by the neutral player), and each node has an amount of troops on it.

b. Deploy

Each player has an amount of troops they get to deploy. The default amount is how many nodes you control, but may vary based on game settings.
You get to choose where to deploy them and how many.
You can split them up however you want, and you can even choose not to deploy all/any of them.
You will, however, not be able to save them up and deploy later. They are lost if you do not use them.

c. Move

Each turn, every player can choose to send a specific amount of troops from any controlled node to an adjacent node (directly connected with a link).

d. Battle

When opposing armies meet, they fight. Each army loses an equal amount of troops until one army remains.
There are two moment where armies battle: 1. While traveling to a node 2. When sieging a node

e. Travel battle by example

Player 1 has node A, player 2 has node B, and node A and B are linked.
Player 1 sends 10 troops from A to B.
Player 2 send 6 troops from B to A.
The armies meet on the link between A and B, and 4 troops are left from player 1.
These 4 troops continue their path to node B.

f. Siege battle by example

Players 1,2 and 3 have nodes A,B and C respectively, and they all have 10 troops.
B is linked to both A and C.
Player 1 sends 8 troops from A to B.
Player 2 stands their ground.
Player 3 sends 4 troops from C to B.
The resulting siege in node B has the following participants: A(8) - B(10) - C(4)
The winner is B with 2 troops remaining, and keeps control of their node.

g. Winning

The goal is to be the only player remaining with nodes under their control.


2. State syntax

In the state your bot receives, the neutral player will have id 0, and your bot will have id 1.
There is a lot of data included that is only used for animations, so don't be intimidated.
The only things that are relevant to your bot probably are:
  • players - to see how many troops everybody gets to deploy
  • nodes - to get the structure of the game itself
  • links - to get the structure of the game itself
  • stages.end - to get the actual latest state

b. JSON example

4. Notes

  • Deploys happen before moving. You can move troops that are freshly deployed.
  • If you have 5 troops on a node, sending 5 troops is allowed, but you will lose control of the node.
